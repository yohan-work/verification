# 02. 프롬프트 엔지니어링 핵심 기법

좋은 질문이 좋은 답변을 만듭니다. LLM의 성능을 200% 끌어올리는 3가지 핵심 기법을 배웁니다.

## 1. Zero-shot vs Few-shot

### Zero-shot (예시 없음)
- 모델에게 아무런 예시 없이 바로 지시하는 것.
- **입력**: "이 영화의 감정을 분류해: '정말 시간 낭비였어.'"
- **단점**: 형식이 제멋대로이거나, 복잡한 뉘앙스를 못 잡을 수 있음.

### Few-shot (예시 제공) ⭐ *강력 추천*
- 모델에게 "이런 식으로 답해"라고 예시(Shot)를 몇 개 주는 것.
- 성능이 비약적으로 향상됩니다.
- **입력**:
    ```text
    문장의 감정을 '긍정' 또는 '부정'으로 분류해.

    예시 1:
    문장: "배송이 너무 빨라서 좋았어요!"
    감정: 긍정

    예시 2:
    문장: "박스가 다 찌그러져서 왔네요."
    감정: 부정

    문제:
    문장: "생각보다 색감이 별로지만 쓸만해요."
    감정:
    ```

## 2. Chain-of-Thought (CoT, 생각의 사슬)

모델에게 **"답만 내놓지 말고, 풀이 과정을 먼저 생각해"**라고 시키는 기법입니다. 논리적 추론 능력이 크게 향상됩니다.

- **Bad Prompt**: "철수가 사과 5개를 샀고, 영희가 2개를 가져갔어. 3개를 더 샀으면 몇 개야?"
- **CoT Prompt**: "철수가 사과 5개를 샀고... (중략) ... **단계별로 생각해서(Let's think step by step)** 답을 구해줘."

### 적용 방법
프롬프트 끝에 **"생각 과정을 서술하고 나서 결론을 말해줘."** 라고 덧붙이세요.

## 3. ReAct (Reasoning + Acting)

AI가 스스로 **생각(Reasoning)**하고 **행동(Acting)**을 결정하는 에이전트의 기초가 되는 기법입니다.
(API나 툴이 연결되어 있지 않은 무료 웹 버전에서는 시뮬레이션만 가능합니다.)

### 구조
1. **Thought**: 사용자의 질문을 분석한다.
2. **Action**: 필요한 행동(검색, 계산 등)을 정의한다.
3. **Observation**: 행동의 결과를 관찰한다.
4. **Answer**: 최종 답변을 한다.

### 실습 프롬프트 예시
```text
질문: 2024년 한국 시리즈 우승팀 감독의 나이는?

너는 검색 도구를 가진 에이전트라고 상상해. 실제 검색은 못하지만, 검색하는 척 시뮬레이션을 해봐.
다음 형식을 지켜:
Thought: (무엇을 알아야 하는지 생각)
Action: (검색 키워드)
Observation: (가상의 검색 결과)
...반복...
Final Answer: (최종 답)
```
